ASP.NET MVC - Student Project

Student Task Management System

Project Overview

Build a simple Student Task Management System using ASP.NET MVC (.NET 9.0). This project covers essential MVC concepts.

Note: Focus on functionality over aesthetics.

Learning Objectives

This project will test your understanding of:

  • MVC Architecture - Models, Views, Controllers
  • Entity Framework Core - Code-First approach
  • Data Annotations and Validation
  • CRUD Operations - Create, Read, Update, Delete
  • LINQ Queries - Filtering and sorting data
  • Routing - Attribute routing with parameters
  • ViewBag - Passing data to views
  • Form Handling - GET and POST methods

Project Requirements

1. Database Models (20 Points)

Create the following models with proper data annotations:

A. Student Model

Property Type Validation
Id int Primary Key, Auto-increment
Name string Required, 3-50 characters
Email string Required, Valid email format
RollNumber string Required, Unique
Department string Required

B. Task Model

Property Type Validation
Id int Primary Key, Auto-increment
Title string Required, 5-100 characters
Description string Required, MultilineText
DueDate DateTime Required, DataType.Date
Priority string Required (Low, Medium, High)
IsCompleted bool Default: false
StudentId int Foreign Key to Student

C. TaskContext (DbContext)

Create a database context with:

  • DbSet for Student
  • DbSet for Task
  • Connection string in appsettings.json

2. Controllers and Actions (30 Points)

A. StudentsController

Action HTTP Method Description
Index() GET Display all students in a table
Create() GET Show form to add new student
Create(Student s) POST Save new student with validation
Edit(int id) GET Load student for editing
Edit(Student s) POST Update student details
Delete(int id) GET Delete student and their tasks

B. TasksController

Action HTTP Method Description
Index() GET Display all tasks with student names
Create() GET Show form to add new task (with student dropdown)
Create(Task t) POST Save new task with validation
MarkComplete(int id) GET Toggle task completion status
Delete(int id) GET Delete a task
StudentTasks(int studentId) GET Show all tasks for a specific student

3. Views (20 Points)

Required Views

View Description Elements
Students/Index.cshtml List all students Table with Name, Email, Roll Number, Department, Actions (Edit, Delete, View Tasks)
Students/Create.cshtml Add new student Form with all student fields, validation messages
Students/Edit.cshtml Edit student Pre-filled form with student data
Tasks/Index.cshtml List all tasks Table with Title, Student Name, Due Date, Priority, Status, Actions
Tasks/Create.cshtml Add new task Form with dropdown for student selection
Tasks/StudentTasks.cshtml Tasks for specific student Table showing all tasks for selected student

4. Database Setup (15 Points)

Migration and Database Creation

Step Command Purpose
1 Add-Migration InitialCreate Create migration files
2 Update-Database Create database and tables
3 Seed Data (Optional) Add 3 sample students and 5 sample tasks

Connection String Configuration

"ConnectionStrings": {
  "DefaultConnection": "Data Source=(localdb)\\MSSQLLocalDB;Initial Catalog=TaskManagementDB;Integrated Security=True"
}

Implement the following queries:

Query Requirements

Query Purpose Implementation
Get all tasks with student info Display tasks with student names Use JOIN or Include()
Filter tasks by student Show tasks for specific student Where() clause with StudentId

Tips for Success

  • Work in order: Models first, then database, then controllers, then views
  • Test each feature immediately after implementing it
  • Use proper naming conventions for clarity
  • Don't forget to call SaveChanges() after database modifications
  • Always check ModelState.IsValid before saving data
  • Use ToList() to execute LINQ queries
  • Keep views simple - focus on functionality
  • Use Bootstrap for quick styling without spending much time
